# Deferred and Rejected Issues

Sprint 12 reviewed the remaining product and release issues against the SRS.
None below is an undisclosed mandatory MVP behavior.

| Issue                                              | Disposition                                   | Basis and release impact                                                                                                                                                     |
| -------------------------------------------------- | --------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Code signing and SmartScreen publisher reputation  | Deferred release operation                    | No certificate is available. The unsigned state is visible, documented, scanned, and recorded in the manifest. It does not block functional RC validation.                   |
| Automatic application updates                      | Deferred by `FR-PACK-007`                     | Requires separate signing, integrity, rollback, network, and privacy design. No updater exists in the RC.                                                                    |
| TeX Live adapter                                   | Deferred implementation under `NFR-COMP-005`  | `CompilerAdapter` preserves the extension boundary; MiKTeX is the supported initial distribution.                                                                            |
| Linux and macOS packages                           | Deferred by product scope                     | Windows 11 x64 remains the only supported platform.                                                                                                                          |
| Hosted, collaboration, or multi-user compilation   | Rejected for this release under `NFR-SEC-013` | Requires a separate SRS, threat model, authority model, protocol choice, and sandboxed execution design.                                                                     |
| OS-level hostile-project sandbox                   | Accepted residual risk                        | The product is explicitly for trusted local projects. Timeout, process cleanup, shell escape policy, output bounds, and trust warnings reduce but do not eliminate TeX risk. |
| Automatic external-file reload, compare, and merge | Deferred enhancement                          | Current version-token conflicts preserve both external data and the unsaved editor buffer without silent overwrite.                                                          |
| ZIP64 and compressed ZIP entries                   | Deferred until project bounds require them    | The classic stored ZIP writer fails explicitly beyond its limits and avoids a new runtime dependency.                                                                        |
| Byte-identical NSIS rebuilds                       | Deferred release-engineering improvement      | The tagged source archive is deterministic and the canonical installer/ASAR hashes are recorded. NSIS packaging metadata may vary across rebuilds.                           |
| Additional clean Windows accounts or VMs           | Deferred cross-environment observation        | The installed suite uses an isolated clean application profile on the supported native Windows host and a separate previous-beta profile.                                    |
| Final branded artwork                              | Deferred polish                               | Current icons are functional placeholders and do not affect security, data integrity, compilation, or accessibility.                                                         |
| MiKTeX update state                                | External prerequisite                         | Users maintain MiKTeX separately. The doctor reports readiness from a real compile rather than claiming package-manager freshness.                                           |
